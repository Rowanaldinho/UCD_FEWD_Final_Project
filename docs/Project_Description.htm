Project Outline

I chose to do a Restaurant website. I looked at the remit for the final project and felt that I could demonstrate all requirements using this type of website.

I set out to create a simple but intuitive website which is easy to navigate and utilises many of the skills I learned throughout the course including

-          The Box Model

-          CSS styling

-          Javascript in the Book A Table page and Menu page

-          Animation in the Our Story page

-          Bootstrap in the Nav Bar and Homepage carousel

-          Node.js. I used the Nodemailer module to send emails from my Book a table page. I used this module in conjunction with Gmail to facilitate the email function on the website. I did initially use SendGrid, but ran into issues due to the limits on the amount of emails available on the free account, so decided to switch to Gmail instead.

-          External REST API. I linked to the Mealdb external API for meals to include on my Menu page

-          I created JSON files in both my Book a table and Menu javascript files.

-          I put an emphasis on accessibility by adding intuitive alt text and aria snippets to my webpages

-          I utilised Figma as a design and illustration tool and have created a Final Project web structure on Figma. I am conscious that Figma is much more powerful than what I used it for and will look to enhance my skills in this area going forward

-          I learned how GitHub works and have been developing my skills in that area. I carried out a large number of commits.

-          I used Canva and Unsplashed to find suitable images for the website

-          I did implement Astro on one of my practice websites, but didnt use it in the Final Project

 

Colour Scheme

I opted for a black and strong pink for my colour scheme and named the restaurant Flavour and Finesse. The colour scheme looked punchy and attractive on the website, and is compliant with double A accessibility contrast requirements.

 

 

Main Theme Colours

Black: rgb (0, 0, 0)

White: rgb(255, 255, 255)

Pink: rgb(241, 10, 191)

Dark Grey on carousel and booking form buttons for better contrast: rgb(34, 34, 34)

 

Nav Bar

The Nav Bar is a Bootstrap nav bar which I have customised with animation to show when a button is in focus. You can also use your keyboard to tab across the buttons. I added styling for the nav bar to the main.css file. This styling applies to all pages on the website for the footer and nav bar.

 

Footer

I used a single container for the footer which I set up with 4 sections using a CSS Grid with 4 grid-template-columns. I used flex-wrap to ensure the footer is responsive. I added relevant text to each column i.e. Restaurant name, Opening hours, Contact details, Address. I added a solid pink border at the top of the footer to separate it from the main body of the webpage.

 

Home Page

-          I utilised a bootstrap carousel to display images of the restaurant and the products and services offered.

 

-          I added buttons to each image on the carousel slides to allow end users to access the booking page or the menu page.

 

-          I set a timer on the images to move the slides at 6 second intervals in order to allow end users to read the text on each slide.

 

-          The end user can stop the carousel by holding the mouse arrow over the slide in focus.

 

-          The slides can be advanced manually by the end user by clicking either the arrow buttons on the slide or the carousel navigation buttons under the buttons

 

-          I added CSS to the buttons on the slides to change their appearance when clicked.

 

Our Story Page

 

-          I utilised a grid layout in the Our Story page in order to allow me better control over where to position the images and text on the page.

 

-          I created 3 rows on the page each divided into columns.

 

-          Rows 1 and 3 use the same structure with the text taking up 3 frames on the left of the row while the image takes up just 1 frame to the right of the row.

 

-          In row 2 the columns are flipped with the image using 1 frame in the left of the column and the text using 3 frames to the right of the image.

 

-          I used animation on this page to add some life to the page when it loads. I set the elements to the left of the page to fade in from the left, while the elements on the right fade in from the right.

 

Menu Page

-          I used a Fetch API to retrieve data from the external MealDB REST API.

 

-          I collect the data retrieved from MealDB into a categories JSON.

 

-          The javascript then dynamically creates a card for each category with an image, heading and description of each menu item.

 

-          The cards are added to the menu element in the main container on the Menu page.

 

-          I also changed the default appearance of the API using CSS to make it fit in with the colour scheme of the website.

 

-          I added a pop-out animation to each card which works when the mouse arrow hovers over each card

 

 

Book a table

-          I created a booking form on the webpage where users must input data for their booking.

-          Once the user hits the submit button, I used a Fetch API on the bookingform.js to capture the data and post it to the backend server i.e. server.js.

 

-          Express.js on server.js receives the data and extracts it. I added a piece of code to the server.js that reformats the American date format to European Date format.

 

-          The booking data including the date in European format is then added to an email.

 

-          I installed nodemailer on the website to handle the sending of emails from the server.

 

-          I configured Gmail to handle delivery of the following emails

-          a confirmation email to the person who submitted the booking containing details of the booking

-          and a second email to go to the restaurant owner containing details of the booking.

 

-          A confirmation message is then displayed on the webpage confirming the booking

 

Contact Us

-          The Contact Us page is the simplest page on the site.

-          The main feature I wanted to demonstrate on this webpage was the inclusion of an iFrame to display a Google Map showing the location of the restaurant.

-          I have not added a specific point on the Google Map as the restaurant is not real. However, the map does display the imaginary location for the restaurant that is linked to the address listed on the website.

 

 

 

Responsiveness

-          The website is fully responsive. The bootstrap features resize according the screen size.

 

-          I have set the grid components in the footer and on the Our Story page to reduce to a stacked single column when the screen is resized.

 

-          I hid column 1 in the Booking Form page when viewing on a smaller screen as the image in column 1 was used for decorative purposes only on the larger screens and was not needed on the smaller screens. I placed emphasis on displaying the booking form.

 

-          On the My Story page, I changed the order of the image and text when I reduced the screen size so that they stacked in a logical order.

 

-          I utilised the flex styling attributes for resizing in the footer container and in the booking form.

 

-          I reduced the font sizes and button sizes when applying styling to the Home screen carousel.
